# V13 / ApplicationV2 Reference

Patterns used by all DCC sheets and dialogs. V13 introduced ApplicationV2; V14 continues to use it. This is the long-lived reference ‚Äî see [V14](V14.md) for V14-specific API changes layered on top.

## Base Class Selection

```
ApplicationV2 (Base)         ‚Üí Configuration dialogs, custom tools
‚îú‚îÄ‚îÄ DocumentSheetV2          ‚Üí Journal entries, other documents
‚îÇ   ‚îú‚îÄ‚îÄ ActorSheetV2         ‚Üí Actor sheets (automatic drag/drop, item/effect handling)
‚îÇ   ‚îî‚îÄ‚îÄ ItemSheetV2          ‚Üí Item sheets (document features, header controls)
‚îî‚îÄ‚îÄ DialogV2                 ‚Üí Modal prompts, confirmations
```

Use `HandlebarsApplicationMixin` for Handlebars templates:

```javascript
const { HandlebarsApplicationMixin } = foundry.applications.api
const { ActorSheetV2 } = foundry.applications.sheets

class DCCActorSheet extends HandlebarsApplicationMixin(ActorSheetV2) { ... }
```

For simple prompts, prefer `DialogV2.confirm()` / `.prompt()` / `.input()` / `.wait()` factory methods.

## Core Class Structure

```javascript
static DEFAULT_OPTIONS = {
  classes: ['dcc', 'sheet', 'actor', 'themed', 'theme-light'],
  tag: 'form',                    // REQUIRED for forms/dialogs
  position: { width: 600, height: 600 },
  window: { resizable: true, title: 'DCC.ActorSheetTitle' },
  form: {
    submitOnChange: true,         // For sheets
    closeOnSubmit: false
  },
  actions: {
    rollAbilityCheck: this.#rollAbilityCheck,
    editImage: DCCActorSheet.editImage  // Use inherited editImage for Forge compatibility
  }
}

static PARTS = {
  tabs:      { template: 'systems/dcc/templates/actor-partial-tabs.html' },
  character: { template: 'systems/dcc/templates/actor-partial-pc-common.html' },
  equipment: { template: 'systems/dcc/templates/actor-partial-pc-equipment.html' }
}

static TABS = {
  sheet: {
    tabs: [
      { id: 'character', group: 'sheet', label: 'DCC.Character' },
      { id: 'equipment', group: 'sheet', label: 'DCC.Equipment' }
    ],
    initial: 'character'
  }
}

async _prepareContext(options) {
  const context = await super._prepareContext(options)
  context.system = this.actor.system
  return context
}
```

**Gotchas:**
- Cannot use `CONFIG.DCC.*` in static field initializers ‚Äî CONFIG isn't populated yet. Hard-code template paths.
- Constructor takes a single options object: `new DCCActorConfig({ document: actor, position: {...} })`, never `new DCCActorConfig(actor, {...})`.
- Access the document via `this.options.document` (or `this.actor` / `this.document` on sheets).

## HTML and DOM

V13+ hooks and `_onRender` receive **plain DOM elements**, not jQuery:

```javascript
// Always:
html.querySelector('.my-button').addEventListener('click', handler)
html.classList.add('active')
html.innerHTML = content
element.dataset.messageId
```

Document updates use `system`, not `data`:

```javascript
this.actor.update({ system: { foo: 1 } })  // not { data: { foo: 1 } }
```

## Actions System (not `activateListeners`)

Map `data-action` attributes in templates to static handlers in `DEFAULT_OPTIONS.actions`:

```html
<span data-action="rollAbilityCheck" data-ability="str">Strength</span>
<button data-action="itemCreate" data-type="weapon">Create Weapon</button>
```

```javascript
static async #rollAbilityCheck(event, target) {
  const ability = target.dataset.ability
  await this.actor.rollAbilityCheck(ability)
}
```

Handlers are bound to the instance (`this`). Additional `data-*` attributes come through `target.dataset`.

## Tabs

Each tab must be its own `PART` (one root element, the `<section>` below). **Do not** use a container template with `<template>` placeholders ‚Äî it breaks focus and state preservation.

```html
<section class="tab {{tabs.character.id}} {{tabs.character.cssClass}}"
         data-tab="{{tabs.character.id}}"
         data-group="{{tabs.character.group}}">
  ...
</section>
```

Tabs nav partial:

```html
<nav class="tabs sheet-tabs" data-group="{{tabs.sheet.group}}">
  {{#each tabs.sheet.tabs}}
    <a class="tab {{this.cssClass}}" data-tab="{{this.id}}" data-group="{{this.group}}">
      {{localize this.label}}
    </a>
  {{/each}}
</nav>
```

Dynamic tabs: override `_getTabsConfig(group)`.

**CSS pitfall:** never set `display: grid` / `display: flex` on the direct parent of `.tab` sections ‚Äî all tabs become visible simultaneously. Move the `display` rule one level deeper.

## Forms

### Templates

With `tag: 'form'`, the framework provides the `<form>` element. Templates must **not** wrap content in `<form>` ‚Äî use `<section>` or `<div>`.

### Inline item edits on actor sheets

ApplicationV2 form validation strips nested item data. For inline weapon/equipment/spell edits, namespace inputs with `items.{id}.property` and override `_processFormData` / `_processSubmitData`:

```javascript
_processFormData(event, form, formData) {
  const expanded = foundry.utils.expandObject(formData.object)
  if (expanded.items) {
    this._pendingItemUpdates = Object.entries(expanded.items).map(([id, data]) => ({ _id: id, ...data }))
  }
  return super._processFormData(event, form, formData)
}

async _processSubmitData(event, form, formData) {
  const result = await super._processSubmitData(event, form, formData)
  if (this._pendingItemUpdates?.length) {
    await this.document.updateEmbeddedDocuments('Item', this._pendingItemUpdates)
    delete this._pendingItemUpdates
  }
  return result
}
```

### Image editing

Use the inherited `editImage` from `DocumentSheetV2` ‚Äî it handles Forge VTT URL edge cases correctly. Template: `<img src="{{img}}" data-action="editImage" data-edit="img">`. Action: `editImage: DCCActorSheet.editImage`.

## Drag and Drop (DCC Hybrid Approach)

`ActorSheetV2` handles basic item/effect drag/drop automatically for elements with `class="draggable"` + `data-item-id` (or `data-effect-id`). DCC overrides this with custom selectors + drag types (ability checks, spell checks, weapons), then delegates back to `ActorSheetV2` for standard item drops.

Key points of the DCC pattern (`module/actor-sheet.js`):

```javascript
static DEFAULT_OPTIONS = {
  dragDrop: [{ dragSelector: '[data-drag="true"]', dropSelector: '.dcc.actor' }]
}

constructor(options = {}) {
  super(options)
  this.#dragDrop = this.#createDragDropHandlers()
}

_onRender(context, options) {
  this.#dragDrop.forEach(d => d.bind(this.element))
}

_onDragStart(event) {
  const dragAction = event.currentTarget.dataset.dragAction
  // Branch on dragAction: 'ability' | 'spellCheck' | 'weapon' | ...
}
```

For non-Actor apps on plain `ApplicationV2`, you must wire up `DragDrop` manually (see `foundry.applications.ux.DragDrop`). `DocumentSheetV2` and `ItemSheetV2` generally don't need drag/drop.

Static helpers used from instance methods must be public (`static foo()`), not private (`static #foo()`).

```html
{{#if editable}}
  <prose-mirror
    name="system.class.corruption"
    button="true"

    toggled="false"
    value="{{system.class.corruption}}">
    {{{corruptionHTML}}}
  </prose-mirror>
{{else}}
  {{{corruptionHTML}}}
{{/if}}
```

Enrich in `_prepareContext`:

```javascript
const { TextEditor } = foundry.applications.ux
context.corruptionHTML = await TextEditor.enrichHTML(
  this.document.system.class.corruption,
  { secrets: this.document.isOwner, relativeTo: this.document }
)
```

## Theming

Variables live in `styles/variables.css` (registered with `"layer": "variables"` in `system.json`). Use the `system-` prefix to avoid colliding with Foundry's own variables. Dark overrides go under `.theme-dark`.

```json
"styles": [
  { "src": "styles/variables.css", "layer": "variables" },
  { "src": "styles/dcc.css",       "layer": "system"    }
]
```

CSS Layers pushed Foundry's defaults into lower layers, so system rules take precedence without `!important`. Checkboxes now render via FontAwesome ‚Äî style `::before` / `::after` and the `:active` state, not the native element.

## Context Menu Hooks

V13 renamed all directory context menu hooks and changed their signature:

| V12 | V13 |
|-----|-----|
| `getChatLogEntryContext` | `getChatMessageContextOptions` |
| `getActorDirectoryEntryContext` | `getActorContextOptions` |
| `getSceneNavigationContext` | `getSceneContextOptions` |
| `getCombatTrackerEntryContext` | `getCombatContextOptions` |
| `getMacroDirectoryEntryContext` | `getMacroContextOptions` |
| `getFolderContext` | `getFolderContextOptions` |

New signature: `(application, menuItems) => { ... }`. Callbacks receive `HTMLElement`, not jQuery ‚Äî read IDs from `element.dataset`.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Dialog won't open | V1 constructor pattern | `new X({ document: actor })` |
| All tabs visible at once | `display: grid/flex` on tab parent | Move display one level deeper |
| Actions not firing | `data-action` mismatch | Value must exactly match `actions` key |
| Form won't submit | Missing `tag: 'form'` or nested `<form>` in template | Add `tag: 'form'`, remove nested form |
| `CONFIG is undefined` | Used in static initializer | Hard-code, or resolve lazily in a getter |
| Drag/drop dead on ApplicationV2 | No manual setup | Wire up `DragDrop` in constructor + `_onRender` |
| Items don't save from actor sheet | Nested item data stripped | Override `_processFormData` / `_processSubmitData` (see Forms section) |
| Theme not applying | Missing `"layer": "variables"` or wrong prefix | Check `system.json` styles, use `system-` prefix |
